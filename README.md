# Cyberiot
Network Analyzer

Используемый промпт находится в файле [prompts](prompts.md)

## Как воспроизвести результаты у себя локально
1. **Клонируйте репозиторий и перейдите в него:**
````git clone git@github.com:FMbic/cyberiot.git && cd cyberiot/````

2. Создайте виртуальное окружение используя python и активируйте его
```python3 -m venv .venv && source .venv/bin/activate```

Более подробно смотрите [документацию python](https://docs.python.org/3/library/venv.html)

3. Для выполнения скриптов понадобятся библиотеки перечисленные в файле requirements.txt (зависимости) чтобы их установить воспользуйтесь командой

``pip install -r requirements.txt``

Все используемые дата сеты находятся в директории Deepseek-V3/inSDN/, в каждом скрипте прописанно его местоположение, менять ничего не нужно

4. Находясь в директории ````cyberiot/```` выберете модель из представленных и пройдите в директорию

5. Выберите скрипт файл с расширением ```.py``` 

a. При работе в IDE запустите файл нажав на кнопку запуска (Зависит от IDE)

b. При работе через консоль запустите файл находясь в директории модели посредством выполнения команды ```python3 filename.py```

## Как происходит обучение моделей:

1. Разделение данных: Данные делятся на обучающую (80%) и тестовую (20%) выборки. Модель никогда не видит тестовые данные до финальной оценки.
Балансировка SMOTE: Для редких классов (например, U2R с 3 образцами) SMOTE создает синтетические примеры, чтобы сбалансировать классы. В вашем случае:

2. Бинарная задача: классы сбалансированы (80k Benign → 80k Attack после SMOTE)
Многоклассовая: все классы доведены до размера самого большого (40k)
Обучение Random Forest и XGBoost:

3. Random Forest: ансамбль деревьев решений. Каждое дерево обучается на случайной подвыборке данных и признаков. Итоговое предсказание — голосование большинства деревьев.

4. XGBoost: градиентный бустинг. Деревья обучаются последовательно, каждое следующее исправляет ошибки предыдущих.

5. Метрика	Что означает	Как интерпретировать
Support - Количество реальных примеров этого класса в тестовой выборке	10,000 Benign, 20,000 Attack
Precision -	Из предсказанных как класс — сколько действительно этот класс	1.00 = все предсказанные атаки — действительно атаки (нет ложных срабатываний)
Recall - Из реальных примеров класса — сколько модель нашла	1.00 = найдены все атаки (нет пропущенных)
F1-score - Гармоническое среднее precision и recall	1.00 = идеальный баланс
Accuracy - Доля правильных ответов от всех примеров	1.00 = 30,000 из 30,000 предсказаны верно

6. Почему такие идеальные метрики?

Это нормально для данного датасета по нескольким причинам:

Признаки очень информативны — CICFlowMeter извлекает 80+ статистических характеристик потоков, которые отлично разделяют нормальный и атакующий трафик
Данные сбалансированы после SMOTE — модель видит равное количество примеров обоих классов
XGBoost и Random Forest — мощные алгоритмы, хорошо работающие с табличными данными
Однако идеальные метрики (1.00) могут указывать на переобучение или утечку данных. Возможные причины:
В обучающую выборку могли попасть очень похожие (почти идентичные) примеры
Признаки могут включать информацию, не доступную в реальном времени (например, Flow ID — удалите её, если есть)
